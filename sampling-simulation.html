Sampling designs to efficiently map flow-dependent drying

Author

Tim Lambert (as part of project with Karli Rogers et al.)

Sampling design problem

Let’s consider a stream reach over which we want to map wet-dry boundaries1 and how they change with flow. Flow permanence surveys are performed by walking along the stream and recording locations at which the stream transitions from wet to dry or vice versa. Each such transect survey gives a snapshot of flow permanence for a particular flow level.

# Define function to generate a wiggly curve
# Arguments:
#   x: input values
#   K: Number of sine components (increase for more wiggle)
f <- function(x, K = 6) {
  amps  <- rnorm(K, sd = 1 / (1:K)) # random amplitude (decreasing amplitudes = smoother)
  phases <- runif(K, 0, 2 * pi) # random phases
  
  # Build the random function
  f_raw <- rowSums(sapply(1:K, function(k)
    amps[k] * sin(2 * pi * k * x + phases[k])))
  
  # Rescale:
  target_min <- -0.1
  target_max <-  0.8
  f_scaled <- (f_raw - min(f_raw)) / (max(f_raw) - min(f_raw))  # [0,1]
  f_scaled <- f_scaled * (target_max - target_min) + target_min # [-0.1,0.8]
  f_trunc <- pmax(f_scaled, 0) # truncate below 0
  return(f_trunc)
}

This document outlines possible ways to select efficient sampling designs. Our objective is to select survey times to coincide with flows that are “informative,” in the sense that after surveying \(n\) times over the course of a field season we obtain an accurate flow permanence map.

Notation

Key variables are defined in Table 1.

Note that flow is measured on a relative basis, using \(y(t) \in [0,1]\) to denote the cumulative probability that discharge is at or below the discharge that is observed at time \(t\). Relative rather than absolute flow is used for two reasons: (1) Flow permanence is especially important in headwaters, where gage-based estimates of absolute flow are uncommon. Flow estimation using time lapse images (USGS Flow Photo Explorer) is well suited to monitoring flow in headwaters but generates only relative estimates of flow. (2) If flow were measured on an absolute scale, error in the flow permanence curve would be dominated by high-flow conditions. Using relative flow weights uncertainty in proportion to the amount of time spent under different flow permanence conditions, which seems reasonable when measuring cumulative biological impacts.2

Uncertainty envelops

How do we define the accuracy of a flow-dependent permanence map3? Let’s focus on the uncertainty in the wet-dry boundary (in \(x\)-\(y\) space) that occurs in the gap between two previously surveyed flow levels, \(y_{i-}\) (lower flow) and \(y_{i+}\) (higher flow). @fig-plot shows the scenario. We know that the flow permanence curve must pass through the points at which the state transitions from wet to dry or vice versa. But beyond that, the curves could fall anywhere within the shaded regions.

The sum of all such shaded rectangles is given by

\[Area = (y_{i+} - y_{i-}) \int_x \left[z(x,y_{i+}) - z(x,y_{i-}) \right] dx \]

where the term inside the integral is 0 if surveys \(i\) and \(j\) had the same state \(z\), and 1 if they did not.

How much is the uncertainty envelop reduced by performing an additional transect survey? Let the new transect \(i\) be taken under some intermediate flow level \(y_i\), whose fractional distance between \(y_j\) and \(y_k\) is \(p=\frac{y_i-y_j}{y_k-y_j}\). Then the expected fractional reduction4 in the area of the uncertainty envelop is \(1−p^2(1−p)^2\).

Putting these two pieces together, we obtain an expression for the expected uncertainty reduction (i.e., its utility \(U\)) of an additonal transect \(i\):

\[ U(i) = \left(1 - p^2 (1-p)^2 \right) (y_{i+}-y_{i-}) \int_x \left[z(x,y_{i+}) - z(x,y_{i-}) \right] dx \]

where \(p\) is a function of the current flow, as defined earlier.

The utility function can be used to guide sampling times. Given a set of completed surveys \(j = (1, 2, ..., i-1)\), the utility function can be calculated for all possible relative flow values \(y\) (Fig. X).

One approach is an adaptive design that utilizes a greedy sampling algorithm . Under greedy sampling, each sample is taken at the maximal value indicates the flow level for which performing an additional transect results in the most improvement in the accuracy of the flow permanence map. Caveats and next steps

  • The greedy algorithm described above optimizes flow level on a sample-by-sample basis, but it lacks any sort of planning beyond the next sample. An alternative is to design a priori an optimal design with \(m\) samples, based on prior knowledge. Previous surveys will give some indication of how the frequency of drying / isolated pools / wet changes with relative flow, although the tradeoff will be that this information is incomplete and outdated (if stream morphology has changed). Perhaps a good compromise would be to do a little of both, e.g., if you wanted to survey each stream a total of five times in 2026, you could use past information to guide an initial \(m = 3\) design, then follow up with two additional “greedy” samples.

  • We haven’t considered the current and future availability of different flow levels. If a particular flow level is uncommon, it may be desirable to sample it even if it is not the planned next sample.

  • This document assumes that surveys are completed in their entirety or not at all, but it could also possible to perform partial transects at a reduced cost/effort. This could be particularly beneficial if early surveys can identify some stream segments for which additional low- or high-flow surveys are not required (e.g., if a segment is wet even during low flow, then there is no need to survey it under higher flow conditions).

  • I’m not sure that I’m using the best language to describe flow permanence and the various stream conditions. Suggested wording changes are welcome.
